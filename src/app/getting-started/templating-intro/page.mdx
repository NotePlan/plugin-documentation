# Getting Started

<Callout
  type="info"
  description="If you haven't yet read the <a href='https://help.noteplan.co/article/136-templates' target='_blank' rel='noopener noreferrer'>Introduction to Templates</a>, please start there."

With `Templating`, you will be able to create powerful templates to automate common manual tasks. For instance, you can use Templating to create your preferred outline in each daily note when it is first opened. Or you can use Templating to create a meeting notes document for each meeting you attend. Templates can be as simple as just text (e.g. headings) you want in a note, or can contain complex Javascript code and do just about anything you can imagine.

## The Basics

The following examples demonstrate how you create a **template** and use `Templating` tags to place dynamic **output**.

### Example 1: Template Definition

When creating a new template in the "Templates" folder, the template header will be in `frontmatter` format, containing two fields for the template--`title` and `type`-- followed by the template body.

`title` - The title is the name of this template (not the name of any notes that will be created from it). You can use the same title in multiple templates, provided they are in different folders. However, it makes it easier to find the right template later if the title is unique and descriptive.

`type` - The type field is used to identify what type of template this is. There are currently 3 different types of templates

- `empty-note` will be available when you select the "Template" button in an empty calendar day
- `quick-note` will be available whenever you invoke the `np:qtn` command
- `meeting-note` will be available whenever you wish to create a meeting note for any NotePlan calendar event
- `template-runner` will be available whenever you use the <Link href="/advanced-features/templating-templateRunner">Template Runner</Link>) command
- `ignore` will be ignored in all template chooser interfaces -- use this for templates you want to keep around but not use right now

For more information on using Quick Notes and the `np:qtn` command, see <Link href="/core-features/templating-quicknotes">here</Link>.

```markdown
---
title: myTemplate
type: empty-note
---

Hello <%- user.first %>
```

```text
Hello Michael
```

### Example 2: Using `Templating` modules

In this example, we will use one of the many `Templating` module methods, <Link href="/built-in-modules/templating-modules-web">Templating Web Module.</Link>.

```markdown
---
title: myTemplate
type: empty-note
---

Today's Weather: <%- web.weather() %>
```

```text
Todays Weather: Fountain Valley, California, United States: ⛅️  +86°F
```

The call to `<%- web.weather() %>` will use the format you have defined in `Templating` <Link href="/configuration/templating-settings">settings</Link>. In this example, the settings value is blank, thus the default weather format will be used.
